Domain Notes

RBAC active

Roles currently present: admin, agent, and customer. Permission counts are admin: 1, agent: 11, customer: 16.

  • roles.name is unique.
  • role_permissions enforces unique (role_id, grpc_method).
  • users.role is text and has no declared FK to roles.name.

Orders active

All current orders are in draft status. orders.order_number is unique.

  • orders.customer_id has a declared FK to customers.
  • orders.user_id is indexed but currently has no declared FK to users.
  • order_services stores denormalized price/name fields for historical line item context.

Customers active

Customers can be attached to users with customers.user_id. A partial unique index enforces one customer profile per non-empty user ID.

Patients legacy

The patients table has no declared foreign keys and currently has zero estimated rows. It appears separate from the current customer/order/service workflow.

Recommended Follow-Ups

RecommendationReason
Consider a declared FK from orders.user_id to users.id.The column is indexed and appears to reference users, but referential integrity is not enforced.
Consider aligning users.role with roles.name.RBAC depends on role names, but the database does not enforce that user roles exist in roles.
Evaluate text date fields.Several scheduling fields are stored as text; using date or timestamptz would improve validation and reporting.
Keep BI/intelligence tables out of this OLTP database.Per BI requirements, business events and AI insights should live in a separate analytics database to avoid reporting load on transactional workflows.
